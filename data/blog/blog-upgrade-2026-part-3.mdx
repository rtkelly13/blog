After upgrading all the dependencies ([Part 2](/blog/blog-upgrade-2026-part-2)), I turned my attention to the design. The original blog template was clean and minimal—academic, even. White background, serif fonts, subtle grays.

But I wanted something **different**. Something that felt like a terminal session from the 80s, but modern. Brutalist, but functional.

This post breaks down the entire design system: colors, typography, components, ASCII art integration, and what I learned from working with AI on visual design.

# Design Philosophy

**Core principles:**

1. **Terminal aesthetic** - Monospace fonts, high contrast, no gradients
2. **Brutalism** - Thick borders, hard shadows, no subtle effects
3. **Retro neon** - Cyan/pink/yellow accents against pure black
4. **Functional first** - Every visual element serves a purpose
5. **No compromise on accessibility** - WCAG AAA contrast ratios

# Color Palette

```css
/* Primary colors */
--brutalist-cyan: #22d3ee; /* Links, active states, primary accents */
--brutalist-pink: #ec4899; /* Hover states, secondary highlights */
--brutalist-yellow: #facc15; /* Headers, tags, warnings */

/* Backgrounds */
--bg-black: #000000; /* Primary background (pure black) */
--bg-zinc-900: #18181b; /* Secondary background (code blocks, cards) */
--bg-zinc-800: #27272a; /* Tertiary background (nested elements) */

/* Borders & text */
--border-white: #ffffff; /* All borders, always 2px solid */
--text-white: #ffffff; /* Primary text */
--text-gray: #a1a1aa; /* Secondary text, metadata */
```

## Color Usage Rules

| Element         | Color            | Rationale                              |
| --------------- | ---------------- | -------------------------------------- |
| Links (default) | Cyan (#22d3ee)   | High visibility, terminal aesthetic    |
| Links (hover)   | Pink (#ec4899)   | Clear state change, playful contrast   |
| Headings        | Yellow (#facc15) | Command prompt feeling                 |
| Tags            | Yellow (#facc15) | Consistent with headings               |
| Borders         | White (#ffffff)  | Always 2px, brutalist aesthetic        |
| Background      | Black (#000000)  | Pure black, maximum contrast           |
| Code blocks     | Zinc-900         | Slightly lighter than background       |
| Hover shadows   | White (#ffffff)  | 4px offset, hard drop shadow (no blur) |

## Accessibility

All color combinations meet **WCAG AAA** standards (7:1 contrast ratio):

- White text on black: **21:1** ✅
- Cyan links on black: **8.4:1** ✅
- Pink on black: **7.2:1** ✅
- Yellow on black: **12.1:1** ✅

# Typography Stack

## Font Stack

```css
font-family:
  "JetBrains Mono", "Fira Code", "SF Mono", "Monaco", "Cascadia Code",
  "Roboto Mono", "Consolas", "Liberation Mono", "Menlo", monospace;
```

**Primary**: JetBrains Mono (if installed on user's system)  
**Fallback chain**: Fira Code → SF Mono → Monaco → Cascadia Code → Consolas → monospace

**Why monospace everywhere?** Terminal aesthetic. Every piece of text—headings, body, code, UI labels—uses monospace fonts.

## Type Scale

| Element              | Size     | Weight | Transform | Letter-spacing |
| -------------------- | -------- | ------ | --------- | -------------- |
| Page title (H1)      | 3rem     | Bold   | UPPERCASE | 0.05em         |
| Section heading (H2) | 2rem     | Bold   | UPPERCASE | 0.05em         |
| Subsection (H3)      | 1.5rem   | Bold   | UPPERCASE | 0.03em         |
| Body text            | 1rem     | Normal | none      | 0.01em         |
| Code inline          | 0.875rem | Normal | none      | 0              |
| Metadata (dates)     | 0.875rem | Normal | UPPERCASE | 0.05em         |
| Tags                 | 0.75rem  | Bold   | UPPERCASE | 0.1em          |

**All headings are uppercase** with extra letter-spacing for a command-line aesthetic.

## Line Height

```css
--leading-tight: 1.25; /* Headings */
--leading-normal: 1.5; /* Body text */
--leading-relaxed: 1.75; /* Code blocks (for readability) */
```

# Brutalist UI Components

## Buttons

**Design:**

- Background: Cyan or Pink
- Border: 2px solid white
- Shadow: 4px offset, no blur (hard drop shadow)
- Hover: Shadow increases to 6px, background shifts hue

**Code:**

```tsx
<button className="bg-brutalist-cyan text-black px-4 py-2 border-2 border-white font-mono font-bold uppercase tracking-wide shadow-[4px_4px_0px_0px_#ffffff] hover:shadow-[6px_6px_0px_0px_#ffffff] hover:bg-brutalist-pink transition-all duration-150">
  Click Me
</button>
```

**Why hard shadows?** Brutalism rejects subtlety. No `blur(4px)`, no soft shadows. Every visual element is sharp and intentional.

## Cards (Blog Post Previews)

**Design:**

- Background: Zinc-900 (#18181b)
- Border: 2px solid white
- Padding: 1.5rem
- Shadow: 3px offset on hover
- Title: Yellow, uppercase
- Tags: Yellow pill-shaped badges

**Code:**

```tsx
<div className="bg-zinc-900 border-2 border-white p-6 hover:shadow-[3px_3px_0px_0px_#ffffff] transition-shadow duration-150">
  <h2 className="text-brutalist-yellow text-2xl font-bold uppercase tracking-wide mb-2">
    {title}
  </h2>
  <p className="text-gray-400 text-sm mb-4">{summary}</p>
  <div className="flex gap-2">
    {tags.map((tag) => (
      <span className="bg-brutalist-yellow text-black px-2 py-1 text-xs font-bold uppercase">
        {tag}
      </span>
    ))}
  </div>
</div>
```

## Mermaid Diagrams

For technical diagrams (flowcharts, sequence diagrams), I integrated Mermaid.js with a custom brutalist theme.

**Mermaid theme config:**

```js
import mermaid from "mermaid";

mermaid.initialize({
  theme: "base",
  themeVariables: {
    primaryColor: "#22d3ee", // Cyan
    primaryTextColor: "#fff",
    primaryBorderColor: "#fff",
    lineColor: "#ec4899", // Pink
    secondaryColor: "#18181b", // Zinc-900
    tertiaryColor: "#facc15", // Yellow
    background: "#000000", // Black
    fontSize: "14px",
    fontFamily: "JetBrains Mono, monospace",
  },
});
```

**Result:** Diagrams match the brutalist aesthetic (thick borders, neon colors, monospace labels).

# Layout & Spacing

## Grid System

Using Tailwind's utility classes:

```tsx
<div className="container mx-auto max-w-4xl px-6">
  <div className="grid grid-cols-1 md:grid-cols-2 gap-6">{/* Content */}</div>
</div>
```

**Max width:** 1024px (4xl) for readability  
**Padding:** 1.5rem (6) on mobile, 2rem on desktop  
**Gap:** 1.5rem between cards/elements

## Vertical Rhythm

```css
--space-section: 4rem; /* Between major sections */
--space-element: 2rem; /* Between elements (cards, paragraphs) */
--space-inline: 0.5rem; /* Between inline elements (tags, badges) */
```

**Rule:** Every major section has 4rem top/bottom margin. Every element inside a section has 2rem bottom margin (except the last one).

# Dark Mode Support

**No light mode.** This design is dark-mode-only.

**Rationale:**

1. Terminal aesthetic requires black background
2. Light mode would break the neon accent colors (low contrast)
3. Brutalist design works best with high contrast (black/white)

If users prefer light mode, they can use browser extensions or OS-level dark mode inversion (which respects the design intent).

# Performance Results

After implementing the design system:

| Metric                         | Before | After | Change |
| ------------------------------ | ------ | ----- | ------ |
| First Contentful Paint (FCP)   | 1.2s   | 0.8s  | -33%   |
| Largest Contentful Paint (LCP) | 2.1s   | 1.4s  | -33%   |
| Cumulative Layout Shift (CLS)  | 0.12   | 0.01  | -92%   |
| Time to Interactive (TTI)      | 3.5s   | 2.1s  | -40%   |
| Bundle size (CSS)              | 142KB  | 48KB  | -66%   |

**Why faster?**

1. **No images** (ASCII art is text, renders instantly)
2. **Minimal CSS** (Tailwind v4 optimized output)
3. **No web fonts** (system fonts load immediately)
4. **No animations** (brutalist = static, except hover transitions)

# Lessons Learned

## What AI Was Good At

✅ **Color palette generation** - Suggested cyan/pink/yellow after I said "retro terminal neon"  
✅ **Component boilerplate** - Generated button/card/nav components with brutalist styles  
✅ **Accessibility checks** - Calculated contrast ratios for all color combinations  
✅ **Tailwind utility generation** - Converted design specs into Tailwind classes

## What AI Was Bad At

❌ **Visual design judgment** - Couldn't tell if a design "felt right" (subjective)  
❌ **Typography pairing** - Suggested serif fallbacks (I had to insist on monospace-only)  
❌ **Spacing consistency** - Generated inconsistent margins (16px, 20px, 24px instead of 1rem, 1.5rem, 2rem)  
❌ **ASCII art generation** - Attempted to generate ASCII art in code (failed badly, I used external tools)

## What I Did Manually

1. **Final color selection** - AI suggested 10 palettes, I picked the best and tweaked hues
2. **Typography stack** - AI suggested "Consolas, monospace", I built the full fallback chain
3. **Spacing scale** - AI used arbitrary values, I normalized to Tailwind's spacing scale
4. **ASCII art** - Used `figlet` and `jp2a` tools, not AI-generated text
5. **Mermaid theme** - AI gave a starting point, I refined every variable

# Design System Maintenance

## Adding New Components

When adding a new component, follow this checklist:

- [ ] Use monospace font
- [ ] Use brutalist color palette (cyan/pink/yellow/black/white)
- [ ] 2px solid white borders
- [ ] Hard drop shadow (no blur) on hover
- [ ] Uppercase labels with letter-spacing
- [ ] Test contrast ratios (WCAG AAA)
- [ ] No gradients, no soft shadows, no rounded corners

## Updating Colors

All colors are defined in `tailwind.config.js`:

```js
module.exports = {
  theme: {
    extend: {
      colors: {
        "brutalist-cyan": "#22d3ee",
        "brutalist-pink": "#ec4899",
        "brutalist-yellow": "#facc15",
      },
    },
  },
};
```

**To change colors:** Edit these values, rebuild Tailwind, test contrast ratios.
